Introduction to Java

Java is a powerful, high-level, object-oriented programming language developed by Sun Microsystems (now owned by Oracle Corporation) in 1995. It is designed to be platform-independent and follows the principle of ‚ÄúWrite Once, Run Anywhere‚Äù (WORA), meaning code written in Java can run on any system that has a Java Virtual Machine (JVM).

How Java Works

Java programs are first written in source files with a .java extension. These files are then compiled by the Java Compiler (javac) into platform-independent bytecode stored in .class files. The Java Virtual Machine (JVM) executes this bytecode by converting it into machine code for the underlying operating system. 

Source Code (.java) ‚Üí Compiler (javac) ‚Üí Bytecode (.class) ‚Üí JVM ‚Üí Machine Code

Key Features of Java

  • Platform Independent: Runs on any OS with a JVM.
  • Object-Oriented: Based on classes and objects.
  • Secure: Provides strong memory management and runs in a sandbox environment.
  • Robust: Has strong error and exception handling features.
  • Multithreaded: Supports concurrent task execution.
  • High Performance: Uses Just-In-Time (JIT) compilation.
  • Portable: Consistent behavior across systems.

Basic Syntax

Every Java program begins with a class definition and a main() method, which serves as the entry point of execution. The main() method is where the Java Virtual Machine (JVM) starts running your program.

Example Program


// Simple Java Program

class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Understanding the main() Method

The line public static void main(String[] args) is special because it tells the Java Virtual Machine where to start program execution. Let‚Äôs break down each part:

  • public: This makes the main() method accessible to the JVM from outside the class. If it‚Äôs not public, the JVM cannot access it and your program won‚Äôt start.
  • static: The method belongs to the class, not to any object. The JVM can call it directly without creating an instance of the class.
  • void: The method does not return any value. The JVM executes it and then ends the program once all instructions are complete.
  • main: This is the predefined method name recognized by the JVM as the starting point of execution.
  • String[] args: This parameter represents an array of strings that stores command-line arguments passed when running the program.

How Command-Line Arguments Work

You can pass additional inputs when running your program from the terminal. For example:


class Example {
    public static void main(String[] args) {
        System.out.println("First argument: " + args[0]);
    }
}

When you compile and run it:


javac Example.java
java Example Hello

The output will be:


First argument: Hello

Handling Missing Arguments

If no arguments are passed, the args array will be empty. Accessing args[0] will cause an error (ArrayIndexOutOfBoundsException). To prevent this, check the array length first:


public static void main(String[] args) {
    if (args.length > 0) {
        System.out.println("First argument: " + args[0]);
    } else {
        System.out.println("No arguments were passed!");
    }
}

Internal Execution Flow

  1. JVM starts and loads the class into memory.
  2. It searches for the method public static void main(String[] args).
  3. It creates a String array and fills it with command-line arguments.
  4. It executes the statements inside main().
  5. After completion, the JVM terminates the program.

Key Takeaways

  • main() is the entry point of all standalone Java programs.
  • public ensures JVM can access it; static allows it to run without objects.
  • String[] args enables passing inputs at runtime.
  • void means it returns no value.

Java Data Types

In Java, data types define the kind of values a variable can store and the amount of memory it will occupy. Java is a strongly typed language, meaning every variable must have a declared data type before use.

üß† Types of Data Types

  • Primitive Data Types: Predefined by Java, used for simple values like numbers and characters.
  • Non-Primitive Data Types: Created by programmers, include classes, arrays, and interfaces.

üíª Primitive Data Types

Data Type Size Example Description
byte 1 byte byte age = 25; Stores small integers (-128 to 127)
short 2 bytes short salary = 10000; Stores larger integers than byte
int 4 bytes int num = 5000; Commonly used integer type
long 8 bytes long distance = 123456789L; Stores very large integers
float 4 bytes float price = 12.34f; Stores decimal numbers (less precision)
double 8 bytes double rate = 19.99; Stores decimal numbers (high precision)
char 2 bytes char grade = 'A'; Stores a single character or ASCII value
boolean 1 bit boolean isValid = true; Stores true or false values

üíª Example Code


class DataTypeExample {
    public static void main(String[] args) {
        int age = 21;
        double salary = 35000.50;
        char grade = 'A';
        boolean isStudent = true;

        System.out.println("Age: " + age);
        System.out.println("Salary: " + salary);
        System.out.println("Grade: " + grade);
        System.out.println("Is Student: " + isStudent);
    }
}

üñ•Ô∏è Output

Salary: 35000.5

Is Student: true

üìã Key Points

  • Primitive types are stored by value and have fixed sizes.
  • Non-primitive types (like Strings) store references, not direct values.
  • Always choose the smallest data type that can hold your value to optimize memory.

Java Control Statements

Control statements determine the flow of program execution. They help make decisions, repeat tasks, and control which parts of the code execute based on certain conditions.

üß† Types of Control Statements

  • Decision-making statements: if, if-else, nested if, switch
  • Looping statements: for, while, do-while
  • Jump statements: break, continue, return

Decision-Making Statements

üí° 1. if Statement

The if statement executes a block of code only when the given condition is true. 


class IfExample {
    public static void main(String[] args) {
        int age = 20;
        if (age >= 18) {
            System.out.println("Eligible to vote");
        }
    }
}

üñ•Ô∏è Output


Eligible to vote

üí° 2. if-else Statement

The if-else statement provides an alternative block of code when the condition is false. 


class IfElseExample {
    public static void main(String[] args) {
        int num = -10;
        if (num >= 0) {
            System.out.println("Positive Number");
        } else {
            System.out.println("Negative Number");
        }
    }
}

üñ•Ô∏è Output


Negative Number

üí° 3. Nested if Statement

A nested if means one if statement inside another. It is used for multiple conditional checks. 


class NestedIfExample {
    public static void main(String[] args) {
        int age = 25;
        String nationality = "Indian";

        if (nationality.equals("Indian")) {
            if (age >= 18) {
                System.out.println("Eligible to Vote in India");
            }
        }
    }
}

üñ•Ô∏è Output


Eligible to Vote in India

üí° 4. switch Statement

The switch statement executes one case from multiple options based on the value of a variable. 


class SwitchExample {
    public static void main(String[] args) {
        int day = 3;
        switch (day) {
            case 1:
                System.out.println("Monday");
                break;
            case 2:
                System.out.println("Tuesday");
                break;
            case 3:
                System.out.println("Wednesday");
                break;
            default:
                System.out.println("Invalid Day");
        }
    }
}

üñ•Ô∏è Output


Wednesday

Looping Statements

üí° 1. for Loop

The for loop executes a block of code a specific number of times. 


class ForLoopExample {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            System.out.println("Count: " + i);
        }
    }
}

üñ•Ô∏è Output

Count: 2
Count: 3
Count: 4
Count: 5

üí° 2. while Loop

The while loop executes repeatedly as long as the condition remains true. 


class WhileExample {
    public static void main(String[] args) {
        int i = 1;
        while (i <= 3) {
            System.out.println("Iteration: " + i);
            i++;
        }
    }
}

üñ•Ô∏è Output


Iteration: 1
Iteration: 2
Iteration: 3

üí° 3. do-while Loop

The do-while loop executes the block at least once before checking the condition. 


class DoWhileExample {
    public static void main(String[] args) {
        int i = 1;
        do {
            System.out.println("Value: " + i);
            i++;
        } while (i <= 3);
    }
}

üñ•Ô∏è Output


Value: 1
Value: 2
Value: 3

Jump Statements

üí° 1. break Statement

The break statement is used to exit from a loop or switch block before it naturally ends. 


class BreakExample {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            if (i == 3) break;
            System.out.println("i = " + i);
        }
    }
}

üñ•Ô∏è Output


i = 1
i = 2

üí° 2. continue Statement

The continue statement skips the current iteration and continues with the next loop cycle. 


class ContinueExample {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            if (i == 3) continue;
            System.out.println("i = " + i);
        }
    }
}

üñ•Ô∏è Output


i = 1
i = 2
i = 4
i = 5

üí° 3. return Statement

The return statement exits a method and optionally returns a value to the caller. 


class ReturnExample {
    public static void main(String[] args) {
        int result = addNumbers(5, 10);
        System.out.println("Sum: " + result);
    }

    static int addNumbers(int a, int b) {
        return a + b; // returns the sum
    }
}

üñ•Ô∏è Output


Sum: 15

üìã Summary

  • if, if-else, and switch handle decision-making.
  • for, while, and do-while handle repetitive tasks.
  • break, continue, and return control the flow within loops and methods.
  • Combining these statements creates logical, dynamic, and flexible programs.
